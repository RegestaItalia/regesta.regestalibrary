(inner) navigateHierarchy(context, hierarchy, optionsopt)

Source:
  • js/NavigationHelper.js, line 155

Executes a router navigation basing on a navigation hierarchy. The hierarchy is an array of navigation routes, note that the route names must coincide with view names.

Parameters:
Name Type Attributes Description
context sap.ui.core.mvc.Controller

The context of the router.
hierarchy array

A collection of routes, sorted as the navigation sequence.
options object <optional>
Properties
Name Type Default Description
direction string Forward

The direction NavigationDirection of the navigation.
parameters object

An object of key/value parameters to carry durig the navigation.
cycle boolean true

A boolean defining if the navigation have to cycle between the hierarchy (e.g. last route will navigate to first route)
beforeNavigateCallback boolean

A callback to be executed before navigation. It can allow or discard navigation by calling resolve or reject functions passed as parameters. It also can override nacigationHierarchy by resolving with the name of a route.
